Control

{@iterate key=obj sort=xxx /}

Iterate over a JavaScript object

Parameters:

  • key : object to iterate over
  • sort : Optional parameter to get sorted output based on the keys. Value can be "asc" for ascending sort, "desc" for descending sort or any other string that defines a user-supplied comparison function for the sort.

iterate provides a mechanism for looping over all the keys of a JavaScript object. Since there is no guaranteed ordering of the keys, you should use the optional sort parameter if you care about portabiity in what you generate.

Within the body of the iterate helper, three values are available:

  • $key - the property name
  • $value - the value of the property
  • $type - the result of typeof on the value

If you want some other compare ordering other than the default string comparison, you can supply a compare function. Here is an example that numerically orders the keys: 


var compareNumbers = function(a,b) {
    var aa= parseInt(a, 10);
    var bb = parseInt(b, 10);
    return aa-bb;
}
{@iterate key=obj sort="compareNumbers"}{$key}:{$value} {/iterate} Your compare function must exist in the dust globals and can be put there by code like this: 

var base = dust.makeBase({"compareNumbers": compareNumbers});
var code = '{@iterate key=obj sort="compareNumbers"}{$key}:{$value} {/iterate}';
dust.render(template, base.push(context), function (err, out) {
    "do something with the output"
  });
